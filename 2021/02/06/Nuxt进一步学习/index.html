扩展路由

nuxt的路由一般是在pages目录里面约定式的生成的，比如pages里面的index.vue文件就会被当成 / 路由匹配的组件，但我们也可以使用扩展路由，即在nuxt.config.js文件的router属性里面对路由进行扩展

router: {
    middleware: 'auth',
    // 扩展路由
    extendRoutes (routes, resolve) {
      console.log(routes)
      routes.push({
        name: 'root',
        path: '/index', // 这里的路由就是 /index 
        component: resolve(__dirname, 'pages/index.vue') // 这里就是pages目录里面的index.vue文件
        // 上面的component引用方式是node的方式，也可使用es的方式 component: ()=>import("@/pages/index.vue")
      })
    }
  },

路由守卫

前置

依赖于nuxt的中间件

有全局中间件（nuxt.config配置middleware ；layout配置；插件配置）和特定中间件（组件独享守卫 middleware ）

特定中间件

middleware(context) {
    console.log('middleware pages');
},

插件的方式

router.js
export default ({ app, redirect }) => {
  console.log('插件前置路由守卫')
  // app是vue的实例
  // redirect是路由跳转函数

  // 下面的这两个都是全局的路由守卫
  app.router.beforeEach((to, from, next) => {
    // 全局前置的守卫
    console.log('vue的钩子进行路由守卫',to)
    // 注意这里的跳转只能由redirect进行，不能由next进行
    if (to.name === 'login' || to.name === 'reg') {
       next()
    } else {
       // 这里的name属性是路由文件的别名，nuxt里面就是文件名，嵌套路由的话需要带目录名 如 home-set
       redirect({ name: 'login' })
       // 或者 redirect('/login')
    }
    next()
  })

  app.router.afterEach((to, from) => {
    console.log('插件全局后置守卫')
  })
}

// 注意插件的方式的话，需要在nuxt.config里面的plugin属性里面引入这个插件文件
plugins: [
    '~plugins/router.js',
  ],

数据交互

就是使用axios进行数据请求，有时候可能需要使用proxy代理一下防止跨域

nuxt-axios官网

一般直接使用nuxt自己集成的axios模块，有时候可能需要安装一下

@nuxtjs/axios @nuxtjs/proxy

在nuxt.confgi里面的modules里面添加axios

modules: [
    '@nuxtjs/axios',
    '@nuxtjs/proxy'
],
axios: {
    proxy: true, // 开启跨域行为
    // prefix: '/api', //baseUrl
},
proxy: { // 代理
// 注意这里是 /api/ 这是nuxt-axios自己规定的方式，有些跟vue-axios不同
'/api/': {
      target: 'http://example.com', //代理转发地址
      changeOrigin: true,
      pathRewrite: {
         '^/api/': ''
      }
    }
},

代理后的使用方式

async asyncData({$axios}){
    // 注意使用的时候也要 使用 /api/ 
    const res = await $axios.get('/api/')
    const { data } = await $axios.get('https://api.chengsong.info')
    // 或者这里直接使用import引入axios进行使用,这里使用了proxy代理
    // const data1 = await axios.get('/api/post', {

    //   url: '/api/post',

    return { res:res,info1: data, info2: data1.data }

    console.log('发送请求时的拦截')
    // config.header.token = '加token'
    // config.headers.token = store.state.user.token
    // return config
  })

  // 响应拦截
  $axios.onResponse((res) => {
    console.log('响应时的拦截')
    // if (res.data.err === 2 && route.fullPath !== '/login') {
    //   redirect('/login?path=' + route.fullPath)
    // }

  // 错误处理

    console.log('出现错误', error)

然后在nuxt.config里面的plugins属性里面引入（注意这里的 ~ 在nuxt默认代表根目录，可在 资源文件 里面看一下使用方式）

plugins: [
    '~plugins/router',
    {
      src: '~plugins/axios',
      ssr: true // 开启服务端渲染，不要这个也没啥大事
    }
  ],

其实感觉吧，这个vuex并没有太多的东西可讲，就一个fetch()方法可能是新的，其他的感觉跟vue-vuex基本没啥大的区别，就简单写一些感觉新的东西吧

1.使用模块的方式 store目录里面的每个js文件都会转换成状态树，index是主模块，其他的就在modules里面

state必须暴露为一个函数，其他的可以是一个对象

// index.js
export const state = () => ({
  counter: 0,
})

export const mutations = {
  increment(state) {
    state.counter++
  },
}

// todos.js
export const state = () => ({
  list: ['默认'],
})

export const mutations = {
  add(state, text) {
    state.list.push({
      text,
      done: false,
    })
  },
  remove(state, { todo }) {
    state.list.splice(state.list.indexOf(todo), 1)
  }
}


// 外部使用
this.info = this.$store.state.todos.list
// 使用子模块的方法
this.$store.commit('todos/add', e.target.value)

状态持久化和token校验

持久化的话，如果使用cookie，需要安装一个包cookie-universal-nuxt ，并在nuxt.config的modules里面引入

modules: [
        'cookie-universal-nuxt',
],

然后使用的方法

const key = "love_home_token"
// 这里的app是vue实例
export function setToken(app, token) {
    app.$cookies.set(key, token);
}

export function getToken(app) {
    return app.$cookies.get(key);
}

token验证的话

一般登录时同步vuex和cookie，如果用户刷新了，nuxtServerInit钩子会被调用，取出cookie，同步vuex，axios拦截器读取vuex（因为vuex存在内存中，所以速度会比存在磁盘的cookie快），设置token

ui组件库的使用

这里以element-ui为例，其实目前nuxt在创建项目的时候就会让你选择一个UI组件库，如果在创建的时候选了，那就直接用就是了，如果没有的话，就使用npm安装一下，在plugins目录里面创建element.js文件（这里是全局引入）

import Vue from 'vue'
import Element from 'element-ui'
import locale from 'element-ui/lib/locale/lang/en'

Vue.use(Element, { locale })

然后在nuxt.config里面的plugins里面引入这个文件并在css里面引入element的css文件

css: [
     'element-ui/lib/theme-chalk/index.css'  // 引入全局样式
],
plugins: [
     {src: '~plugins/element', ssr: true } // ssr:true代表在服务端渲染,客户端不渲染
],

如果需要按需引入的话，需要先安装一个插件 babel-plugin-component 然后在上面的element.js文件里面按需引入

同时如果需要在客户端的时候不引入element，则需要对build进行配置

// nuxt.config.js
build: {
    transpile: [/^element-ui/],
  },

plugins混入

就是提一下这个用处，当在对vue实例进行插入的时候，既然我们都拿到Vue实例了，那所有的vue自己的关于全局的东西都能被使用，如全局过滤器（Vue.filter），全局指令（Vue.directive），全局组件（Vue.component），混入mixin（Vue.mixin）这些都能被使用

// 全局方法

import Vue from 'vue'
let show = () => console.log('全局方法')

Vue.prototype.$show = show // 服务端钩子内部不可以使用  服务端this不会指向vue实例

// 全局过滤器
import * as filters from '../assets/js/filter.js'

Object.keys(filters).forEach(key=>Vue.filter(key, filters[key]))

// 全局指令
import direc1 from '../assets/js/directives/directive1.js'
import direc2 from '../assets/js/directives/directive2.js'
Vue.directive('direc1', direc1)
Vue.directive('direc2', direc2)


// 全局组件
import UcButton from '../components/global/uc-button.vue'
Vue.component('uc-button', UcButton)

// 混入methods
Vue.mixin({ // 都会有这个
  methods: {
    $seo(title, content, payload=[]) {
      return {
        title,
        meta: [{
          hid: 'descript',
          name: 'keywords',
          content
        }].concat(payload)
      }
    }
  }
})

meta信息注入

meta简单的说就是一个关于这个网站的主要信息的总结，是说这个网站是在干什么的

为了爬虫爬取到更多的网页的信息，需要自己定义一下meta里面的内容

实际上nuxtjs是使用了vue-meta 更新应用的 头部标签(Head) 和 html 属性

对于head可以在nuxt.config里面进行统一配置，也可以在一个单独的页面进行特定的配置

// 组件内部方式
head() {
    return {
      title: '晟松',
      meta: [
        {
          hid: 'cs',
          name: '晟松',
          content: this.title,
        },
      ],
    }
  },

// nuxt.config
module.exports = {
  head: {
    titleTemplate: '同一标题',
    meta: [
      { charset: 'utf-8' },
      { name: 'viewport', content: 'width=device-width, initial-scale=1' },
      { hid: 'description', name: 'description', content: 'Meta description' }
    ],
    // 还可以在这里引入一些js脚本，字体文件之类的
      link: [
      { rel: 'icon', type: 'image/x-icon', href: '/favicon.ico' }
    ],
    script: [
      {src: '外部链接'}
    ],
  }
}

资源路径

主要是有时候比如 img 或者其他的什么东西，可能需要找到本地的一个资源，这个时候就需要对路径进行指定

相对路径 找到一些需要压缩资源 assets目录 <img src=”../assets/xxx.png” alt=””>

绝对路径 找到无需压缩的资源 static目录 <img src=”/static/bg.jpg” alt=””> 注意这里使用 / 开头

~assets ~代表根

@代表 src

typescript支持

可以在使用nuxt项目创建的时候就选择使用typescript，也可以其他方式

先安装 @nuxt/typescript-build 然后在nuxt.config里面的引入

export default {
  buildModules: ['@nuxt/typescript-build']
}

之后再建立tsconfig.json 文件

{
  "compilerOptions": {
    "target": "ES2018",
    "module": "ESNext",
    "moduleResolution": "Node",
    // 下面的skip.. 是在一个项目里面新使用ts的时候才需要配置的
     "skipLibCheck":true,
    "lib": [
      "ESNext",
      "ESNext.AsyncIterable",
      "DOM"
    ],
    ....
}
