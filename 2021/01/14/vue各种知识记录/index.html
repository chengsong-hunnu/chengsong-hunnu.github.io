setup集合

根据官网的说明，setup应该是在beforecreated前被调用

下表包含如何在 setup () 内部调用生命周期钩子：

选项式 API Hook inside setup
beforeCreate Not needed*
created Not needed*
beforeMount onBeforeMount
mounted onMounted
beforeUpdate onBeforeUpdate
updated onUpdated
beforeUnmount onBeforeUnmount
unmounted onUnmounted
errorCaptured onErrorCaptured
renderTracked onRenderTracked
renderTriggered onRenderTriggered

TIP

因为 setup 是围绕 beforeCreate 和 created 生命周期钩子运行的，所以不需要显式地定义它们。换句话说，在这些钩子中编写的任何代码都应该直接在 setup 函数中编写。

setup(props,ctx){
    // props是响应式的，但是不能直接解构，会失去响应式，所以需要使用toRefs进行解构
    // ctx有3个属性，attrs,slots,emit
}

执行 setup 时，组件实例尚未被创建。因此，你只能访问以下 property：

  • props
  • attrs
  • slots
  • emit

换句话说，你将无法访问以下组件选项：

  • data
  • computed
  • methods

config
const app = Vue.createApp({})

app.config = {...}

主要是globalProperties

添加可以在应用程序内的任何组件实例中访问的全局 property。属性名冲突时，组件的 property 将具有优先权。

这可以代替 Vue 2.x Vue.prototype 扩展：

// 之前(Vue 2.x)
Vue.prototype.$http = () => {}

// 之后(Vue 3.x)
const app = Vue.createApp({})
app.config.globalProperties.$http = () => {}

还有一些其他的用法，看官网吧

vue-router的特殊用法

一般的嵌套路由是这样的

{
    path: "/components",
    name: "Components",
    component: () => import("@/views/NavView/Components.vue"),
    redirect: "/components/button",
    children: [
        {
            path: "button",
            name: "Button",
            component: () => import("@/views/Button.vue"),
        },
        {
            path: "icon",
            name: "Icon",
            component: () => import("@/views/Icon.vue"),
        },
    ],
},

// 后面的页面路由
/components/button

// 其<router-view />是在Components.vue文件里面

现在看见了一种新的方式

{
    path: "/home",
    component: Home,
    // redirect: "/welcome",
    children: [
        { path: "/welcome", component: Welcome },
        { path: "/less", component: Less },
        { path: "/variable", component: Variable },
        { path: "/mixins", component: Mixins },
        { path: "/nestedRules", component: NestedRules },
        { path: "/operations", component: operations },
        { path: "/namespace", component: Namespace },
        { path: "/important", component: Important }
    ]
},

// 后面的页面路由
/less

// 其<router-view />是在Home.vue文件里面

直接就通过 / 变为绝对地址，就不需要加上前面的那个 /home 的前缀了
