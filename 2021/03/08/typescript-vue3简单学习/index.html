typescript

有些久没用过ts了，之前的知识点都忘了，现在再简单的学一遍吧

首先安装

npm install typescript -g

可使用 tsc 命令查看是否安装完成

可使用

tsc demo.ts

编译ts程序，生成js程序，再使用node跑js程序，但效率太低，可安装 ts-node 使用

ts-node demo.ts

直接跑ts程序

类型

就是定义了一个类型后，就只能对这个变量赋这个类型的值，当类型不匹配的时候，就会报错。并且这个变量也继承了这个类型上面的属性和方法

静态类型

let isDone: boolean = false;
let name: string = `Gene`;
let age: number = 37;

其声明类型的方式就如上面所示

对象类型

let test1: { name: string, age: number } = {
    name: 'test1',
    age: 20
}

类 类型

class Person {
    say() {
        console.log('person');
    }
}
let my:Person = new Person()

以一个类为一个变量的类型，则这个变量只能是这个 类的实例

数组

有两种方式可以定义数组。 第一种，可以在元素类型后面接上 []，表示由此类型元素组成的一个数组：

let list: number[] = [1, 2, 3];
let list1: string[] = ['cs','j']

第二种方式是使用数组泛型，Array<元素类型>：

let list: Array<number> = [1, 2, 3];

函数类型

let test2: () => string = () => { return 'string' }

只能是一个函数，并且还确定了函数的返回值必须是一个string类型

元组 Tuple

元组类型允许表示一个已知元素数量和类型的数组，各元素的类型不必相同。 比如，你可以定义一对值分别为 string和number类型的元组。把数组中的每个元素类型的位置给固定住了，这就叫做元组，虽然一般只会在csv数据格式的时候用到

let x: [string, number];
x = ['hello', 10]; // OK
x = [10, 'hello']; // Error
x = [10, 'hello',12]; // Error，不能越界，只能有两个值

当访问一个越界的元素，会使用联合类型替代：

x[3] = 'world'; // OK, 字符串可以赋值给(string | number)类型

console.log(x[5].toString()); // OK, 'string' 和 'number' 都有 toString

x[6] = true; // Error, 布尔不是(string | number)类型

接口

可以当做一个自定义静态类型，用来规范类型

interface cs {
    uname: string,
    age: number
}
let jcs: cs = {
    uname: '蒋晟松',
    age: 15
}
let sc: cs[] = [{uname:'cs',age:21}]
console.log('jcs: ', jcs); // jcs:  { uname: '蒋晟松', age: 15 }

还可对一个接口里面的属性可选，因为有些属性不全是必须的

可选属性的好处之一是可以对可能存在的属性进行预定义，好处之二是可以捕获引用了不存在的属性时的错误

interface SquareConfig {
  color?: string;    // 加一个?号
  width?: number;
}

function createSquare(config: SquareConfig): {color: string; area: number} {
  let newSquare = {color: "white", area: 100};
  if (config.color) {
    newSquare.color = config.color;
  }
  if (config.width) {
    newSquare.area = config.width * config.width;
  }
  return newSquare;
}

let mySquare = createSquare({color: "black"});

还可使用 redonly 进行指定变量不能变化

interface Point {
    readonly x: number;
    readonly y: number;
}
et p1: Point = { x: 10, y: 20 };
p1.x = 5; // error!

当然在我们的对象里面出现接口里面没有定义的属性的时候，会提示错误，所以可以在接口里面添加一个any类型的值

同时接口里面也能存一个方法，所以相应的对象也需要有一个方法

interface gril {
    name: string,
    age: number,
    height?: number,
    [propname: string]: any    // 属性的名字是字符串类型，属性的值可以是任何类型。
    say(): string;
}
let my4: gril = {
    name: 'cs',
    age: 12,
    cs: '323',
    say() { return '21' }
}

同时接口也可以与类结合，注意这几个属性的写法，特别是name和age是使用 = 进行赋值的，之后可以在实例中使用

class my5 implements gril {
    name = 'cs'    
    age = 45
    say() {
        console.log(this.name);
        return '78'
    }
}

接口之间也可进行继承

interface Teacher extends Girl {
  teach(): string;
}

枚举

enum类型是对JavaScript标准数据类型的一个补充。 像C#等其它语言一样，使用枚举类型可以为一组数值赋予友好的名字。

enum Color {Red, Green, Blue}
let c: Color = Color.Green;

其里面的值默认从0开始，同时也可以对第一个值赋1，让其从1开始

enum里面的值都可以赋值，当然最好赋同一类型的值，防止奇怪的错误

enum en { book = 'cs', water = 3, app = 8 }
let entest: en = en.book    // cs

Any

有时候，我们会想要为那些在编程阶段还不清楚类型的变量指定一个类型。 这些值可能来自于动态的内容，比如来自用户输入或第三方代码库。 这种情况下，我们不希望类型检查器对这些值进行检查而是直接让它们通过编译阶段的检查。 那么我们可以使用 any类型来标记这些变量：

let notSure: any = 4;
notSure = "maybe a string instead";
notSure = false;

类型注解、类型推断

类型注解其实就是对一个变量定义了一个类型，当我们需要这个变量的类型的时候，编译器就会提示我们这个变量是什么类型

类型推断就是不对一个变量声明类型，直接进行赋值，ts会自动推断这个变量应该是什么类型

这两个的用处就是，当ts能分辨一个变量的类型是什么的时候，我们就不要声明类型；当不能分辨的时候，就需要我们进行类型注解

函数参数和返回类型注解

function func(a: string = '21', b: number = 45): string {    
    let s: string = '32'
    let k: string = '14'
    return s + k + a + b
}
func('cs', 21)

也可对函数里面的参数指定类型和默认值

当函数没有返回值的时候，可指定一个void类型，当函数里面有return语句的时候就会报错

function func1(): void {
    console.log('void');
}

当函数无法返回的时候，即抛出错误或者出现死循环，可指定一个never类型

function errfunc(): never {
    throw new Error()
}

当函数的参数是对象的时候，如果需要对类型进行注解，需要

function add({ a, b }: { a: number, b: number }): number {
    return a + b
}
let test4 = add({ a: 21, b: 65 })

虽然这样写会很麻烦，但有一定的规则还是好一些的

特殊类型数组

当数组里面有多种类型的时候

let arr1: (number | string)[] = [1, 2, 3, 'cdl']

当数组中有对象类型的时候，直接进行定义就会看着很麻烦，所以有一个东西叫 类型别名：type 

const xiaoJieJies: { name: string, age: Number }[] = [
  { name: "刘英", age: 18 },
  { name: "谢大脚", age: 28 },
];

type Lady = { name: string, age: Number };
const xiaoJieJies: Lady[] = [
  { name: "刘英", age: 18 },
  { name: "谢大脚", age: 28 },
];

当然使用类进行定义也可以

class Madam {
  name: string;
  age: number;
}

const xiaoJieJies: Madam[] = [
  { name: "刘英", age: 18 },
  { name: "谢大脚", age: 28 },
];

类型别名与接口

类型别名可以直接给类型，比如string，而接口必须代表对象。

比如我们的类型别名可以写出下面的代码：

type Girl1 = stirng;

但是接口就不能这样写，它必须代表的是一个对象，也就是说，你初始化girl的时候，必须写出下面的形式.

interface Girl {
  name: string;
  age: number;
  bust: number;
}

const girl = {
  name: "cs",
  age: 18,
  bust: 94,
};

类

感觉没多大差别

class greet {
    hei = 'cas'
    wid: string    // 在ts里面可以先指定一个属性的类型，但不确认其值是否应该在实例里存在
    constructor(width: string) {
        this.wid = width
    }
    say() {
        console.log(this.hei);
    }
}
let my8 = new greet('87')
console.log('my8: ', my8); // my8:  greet { hei: 'cas', wid: '87' }

继承

ts里面类的继承感觉跟js里面的差别不大，依旧使用 extends 关键字

class Animal {
    name: string;
    constructor(theName: string) { this.name = theName; }
    move(distanceInMeters: number = 0) {
        console.log(`${this.name} moved ${distanceInMeters}m.`);
    }
}

class Snake extends Animal {
    constructor(name: string) { super(name); }
    // 注意这里重写了父类的move方法
    move(distanceInMeters = 5) {
        console.log("Slithering...");
        super.move(distanceInMeters);
    }
}

class Horse extends Animal {
    constructor(name: string) { super(name); }
    move(distanceInMeters = 45) {
        console.log("Galloping...");
        super.move(distanceInMeters);
    }
}

let sam = new Snake("Sammy the Python");
let tom: Animal = new Horse("Tommy the Palomino");

sam.move();
tom.move(34);

重写

其实上面的代码继承就已经使用了重写了

修饰符

同时ts里面的类的属性也具有 public、private、protect 这种修饰符，同时readonly也是可以在上面使用的。默认所有属性都是public

public 允许在类的内部和外部被调用.

private 访问属性的意思是，只允许在类的内部被调用，外部不允许调用

protected 允许在类内及继承的子类中使用

class Animal {
    private name: string;
    readonly age:number;
    constructor(theName: string) { this.name = theName; }
}

new Animal("Cat").name; // 错误: 'name' 是私有的.

构造函数

class greet {
    hei = 'cas'
    private wid: string
    constructor(width?: string) {
        this.wid = width
    }
    say() {
        console.log(this.hei);
        return 78
    }
}
let my8 = new greet('87')

class greet1 extends greet {
    constructor() {
        super('99')    // 依旧要符合规则的使用super调用父类的构造函数，这里就是里面的参数必须是一个string类型
    }
    say() {
        console.log('this.wid:');
        return 12
    }
}

getter和setter

主要是配合private修饰符进行使用，先将一个属性标位private，然后暴露一个getter和setter方法，之后就能预先进行处理或者做些其他的东西

class greet1 extends greet {
    constructor(private _hello: string) {
        super(_hello)
    }
    say() {
        console.log('this.wid:');
        return 12
    }
    get hello() {    // 这只是一个属性，不是一个方法
        return this._hello+'2'
    }
    set hello(hello: string) {
        this._hello = hello
    }
}
let my9 = new greet1('hello1')
console.log('my9: ', my9.hello);    // hello12 ，注意这里的调用方式

抽象类

抽象类做为其它派生类的基类使用。 它们一般不会直接被实例化。 不同于接口，抽象类可以包含成员的实现细节。 abstract关键字是用于定义抽象类和在抽象类内部定义抽象方法。

abstract class abs {
    abstract sayhi(): string;
    move(): void {
        console.log('hi');
    }
}
class abs1 extends abs {
    sayhi() {
        return '78'
    }
}
let my10 = new abs1()
console.log('my10: ', my10.sayhi()); // 78

把类当做接口使用

类定义会创建两个东西：类的实例类型和一个构造函数。 因为类可以创建出类型，所以你能够在允许使用接口的地方使用类。

class Point {
    x: number;
    y: number;
}

interface Point3d extends Point {
    z: number;
}

let point3d: Point3d = {x: 1, y: 2, z: 3};

tsconfig.json

用来配置如何对ts文件进行编译

可使用tsc --init 生成配置文件

配置了配置文件之后，可通过 tsc 命令运行配置文件和此目录里面的ts文件

选择编译文件

"files"指定一个包含相对或绝对文件路径的列表。"include"和"exclude"属性指定一个文件glob匹配模式列表。 支持的glob通配符有：

  • * 匹配0或多个字符（不包括目录分隔符）
  • ? 匹配一个任意字符（不包括目录分隔符）
  • **/ 递归匹配任意子目录
{
    "files": ["demo1.ts"],
    "include": [    // 包含的文件
            "src/**/*"
        ],
    "exclude": [    // 不包含的文件
            "node_modules",
            "**/*.spec.ts"
    ]
}

使用"include"引入的文件可以使用"exclude"属性过滤。 然而，通过 "files"属性明确指定的文件却总是会被包含在内，不管"exclude"如何设置

compilerOptions属性

这个就是这个配置文件里面的配置属性

ts-node命令遵循此配置文件

removeComments 表示是否删除文件里面的注释

strict 是都使用严格模式

noImplicitAny属性的作用是，允许你的注解类型 any 不用特意表明，即可以隐去any，在严格模式下也不会报错

strictNullChecks设置为false,它的意思就是，不强制检查 NULL 类型。

outDir 指定编译后的js文件存放的位置

联合类型和类型保护

联合类型，可以认为一个变量可能有两种或两种以上的类型

interface Waiter {
  anjiao: boolean;
  say: () => {};
}

interface Teacher {
  anjiao: boolean;
  skill: () => {};
}

function judgeWho(animal: Waiter | Teacher) {}

但有时候在一个函数里面不能判断这个变量是哪种类型，该用那些方法，所以就需要类型断言

类型断言就是通过断言的方式确定传递过来的准确值

function judgeWho(animal: Waiter | Teacher) {
  if (animal.anjiao) {
    (animal as Teacher).skill();
  }else{
    (animal as Waiter).say();
  }
}

还有一些其他的方式，比如 *in/ typeof/ instanseof/ *

函数泛型

一种对代码的重用方式

function join(first: string | number, second: string | number) {
    return `${first}${second}`;
}
join("chengsong", ".info");

当我们需要第一个参数的类型为string的时候，第二个参数的类型也要是string，这就需要泛型了

泛型的定义使用 <>（尖角号）进行定义

function join<duanyan>(first: duanyan, second: duanyan) {
    return `${first}${second}`;
}
join<string>("chengsong", ".info");

泛型数组

function al<T>(params: T[]) {
    console.log('params: ', params);
}
al<number>([32, 32])

多泛型定义

function al<T, P>(params: T[], param2: P) {
    console.log('params: ', params);
}
al<number, string>([32, 32], '43')

当然，泛型也可以进行推断，但一般不进行使用，因为代码的可读性和健壮性都会降低

类中泛型

class SelectGirl<T> {
  constructor(private girls: T[]) {}
  getGirl(index: number): T {
    return this.girls[index];
  }
}

const selectGirl = new SelectGirl<string>(["大脚", "刘英", "晓红"]);

泛型继承

interface gril3 {
    name: string
}
class segr<T extends gril3>{
    constructor(private girls: T[]) { }
    getGirl(index: number): string {
        return this.girls[index].name;
    }
}
let selectgr = new segr([{ name: 'cs' }, { name: 'js' }])

这段代码的意思，就是我们在SelectGirl类中使用了泛型，意思是我不知道我以后要用什么类型，但是我有一个约束条件，这个类型，必须要有一个name属性

泛型约束

当一个泛型只能是number或string的时候，就需要对这个泛型进行约束，就是继承所需要的几种类型

class SelectGirl2<T extends string | number> {
    constructor(private girls: T[]) { }
    getGirl(index: number): T {
        return this.girls[index];
    }
}

命名空间 namespace

简单的说，这个就是减少了全局变量的可能带来的错误

一般可以使用 export 导出模块，这样就只有导出的那一个名字是全局的

namespace Shapes {    // 使用namespace进行定义
    export namespace Polygons {
        export class Triangle { }
        export class Square { }
    }
}
