webpack简单使用
2020-09-07
Word count: 442|Reading time: 1 min

就是在菜鸟的一个简单学习和使用，并没有深入

1.首先进行一个webpack的安装
cnpm install webpack webpack-cli -g

打包使用

webpack filename -o targetname --mode=developmention/production  
webpack ./src/index.js -o ./dist/bundle.js --mode=production //例

开发环境里面会包含注释这些，会更详细。生产环境或对代码进行压缩，删除注释，代码混淆这些，大小也更小。

2.webpack.config.js
const path = require('path');//引入node的path
const HtmlWebpackPlugin = require('html-webpack-plugin'); // 通过 npm 安装，生成一个html文件，包含bundle.js
const webpack = require('webpack'); // 用于访问内置插件

module.exports = {
    entry: "./main.js", //入口文件
    output: {
        path: path.resolve(__dirname, 'dist'),, //输出路径，默认为dist文件夹
        filename: "bundle.js" //输出文件名
    },
    mode:'develpoment',//模式
    //loader配置
    module: {
        //先进行文件匹配，在进行处理
        loaders: [
            { test: /\.css$/,
              use:[
                  //use的数组内容loader是从下到上，逆序执行
                  "style-loader", //将js的样式内容插入到style标签里面
                  "css.loader" //将css文件转换成js文件
              ]
            }
        ]
    },
    //插件配置
    plugins: [
    new HtmlWebpackPlugin({
        template: './src/index.html'
    })
  ],
    devServer:{
        contentBase:path.resolve(__pathname,"dist"), //目标路径
        compress:true, //自动刷新
        port:3000  //端口号
    }
};

之后直接在命令行使用

webpack

就可以进行打包

3.loader

loader 让 webpack 能够去处理那些非 JavaScript 文件（webpack 自身只理解 JavaScript）。

4.plugins

使用一个插件，你只需要 require() 它，然后把它添加到 plugins 数组中.在一个配置文件中因为不同目的而多次使用同一个插件，这时需要通过使用 new 操作符来创建它的一个实例

5.热更新

可以在进行修改之后，马上看见webpack后的效果，不需要再进行webpack命令。使用devServe，需要安装依赖。

cnpm install webpack-dev-server -g
webpack-dev-server //进行打开
